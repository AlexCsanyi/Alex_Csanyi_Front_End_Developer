What is Big O notation, and why is it useful?

What is the Big O Notation? In simple terms, it is a mathematical notation we use to describe the complexity of an algorithm. We use it for talking about how long an algorithm takes to run. With it, we can compare the efficiency of different solutions to a problem. With Big O Notation we express the runtime in terms of — how quickly it grows relative to the input, as the input gets larger. Run times grow at different speeds. Therefore it is very important to know how the running time increases as the list size grows.

If you have a list of size n and you perform a simple search you would need to check each element, so it will take n operations to go through the list. The run time in Big O Notation is O(n). It will not tell you the speed of the operation in seconds it will allow you to compare the number of operations. It looks at the worst-case scenario. For example if you have to look for an id in your database and you are looking for the id '0' - this would be the first id in your list. This means your algorithm only had to look at the first entry. However this doesn't mean that the Big O notation will be O(1) time - it is still O(n) time. It works as an assurance - you know that in worst case your algorithm will take O(n) time to run.
